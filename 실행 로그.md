| :----: | -------------- | ----------- | ----------- | ------------------------------------------ |
|  **1** | 카트라이더 AI 설계    | 분석가         | 간결한 분석 톤    | 기본 역할 및 방향 설정                              |
|  **2** | 응답 스타일 조정      | 분석가         | 간결하고 실용적인 톤 | 플레이어 관점의 응답 허용                             |
|  **3** | 안전장치 설정        | 검증형 도우미     | 확인 중심       | 모르면 명시하고 필요한 경우 추가 질문 수행                   |
|  **4** | 답변 방식 정리       | 근거 제시형 분석가  | 짧고 명확       | 근거 제시 및 3가지 요약 규칙 적용                       |
|  **5** | 환각·추정 처리       | 검증 보조 도우미   | 엄격한 톤       | 환각·창작·추정 구분 규칙 적용                          |
|  **6** | 프롬프트 구분        | 이해 보조 도우미   | 설명형         | System Prompt와 User Prompt 분리              |
|  **7** | 기본 정보 정리       | 지식 정리형 튜터   | 설명형         | 역사, 우승자, 리그 구조 정리                          |
|  **8** | Few-shot 학습 적용 | 예시 기반 설계자   | 패턴 설명형      | 질문 → 재질문 패턴 기반 예시 학습                       |
|  **9** | 환각 판단 기준표 제작   | 검증 표 작성 도우미 | 표 중심        | 환각 판단 기준 및 검증 형식 표준화                       |
| **10** | 입력 템플릿 개선      | 구조화 도우미     | 정리형         | 입력 템플릿 구조화 및 문맥 유지 고려                      |
| **11** | 문맥 유지 조건 점검    | 검증형 정리 도우미  | 판정 중심       | 이전 10턴의 합의사항 유지 여부 확인 및 문맥 유지 PASS/FAIL 판정 |

📊 최종 변화 요약

| 구분              | 초기       | 최종                                 |
| --------------- | -------- | ---------------------------------- |
| **AI 역할(Role)** | 분석가      | 검증·정리 중심 도우미                       |
| **응답 톤(Tone)**  | 간결한 분석 톤 | 구조화·검증 중심의 설명 톤                    |
| **프롬프트 구성**     | 단일 프롬프트  | System Prompt / User Prompt 분리     |
| **검증 규칙**       | 기본 응답 중심 | 근거 제시, 환각·추정·창작 구분 적용              |
| **입력 방식**       | 문장형 입력   | 구조화된 입력 템플릿 적용                     |
| **문맥 유지**       | 별도 기준 없음 | 이전 대화 내용 유지 및 일관성 검증(PASS/FAIL) 적용 |
